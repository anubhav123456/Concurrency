# âœ… **What is Dynamic Method Dispatch?**

Dynamic Method Dispatch is the mechanism by which a **call to an overridden method** is resolved **at runtime**, not compile time.

It allows Java to pick the **correct version of an overridden method** based on the **actual object type**, even if the reference is of a parent class.

---

# ðŸ”¥ **Why do we need it?**

* To achieve **runtime polymorphism**
* To allow **flexible and extensible code**
* To write code that works on **parent types but executes child behavior**

---

> Java always decides **the method to run based on the actual objectâ€™s class**, not the reference variable type.

# ðŸ“ **Example**

```java

    void sound() {
        System.out.println("Animal makes a sound");
    }
}

class Dog extends Animal {
    @Override
    void sound() {
        System.out.println("Dog barks");
    }
}

class Cat extends Animal {
    @Override
    void sound() {
        System.out.println("Cat meows");
    }
}

public class Main {
    public static void main(String[] args) {
        Animal obj;      // reference variable of parent class

        obj = new Dog(); // object of Dog
        obj.sound();     // Output: Dog barks

        obj = new Cat(); // object of Cat
        obj.sound();     // Output: Cat meows
    }
}
```
