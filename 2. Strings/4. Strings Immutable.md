## ‚úÖ **Muatability**
* Strings are immutable while StringBuffer and StringBuilder are mutable.
```java
String s = new String("Hello");
s.concat(" World!");
System.out.println(s);
```
* `new String("Hello");` creates a new String Object in heap.
* `s.concat(" World!");` creates another String object in heap as Strings are immutable. 
* Since, `s` is reference of first String Object therefore `System.out.println(s);` will print out `‚ÄúHello‚Äù`.

```java
String s1 = new String("hello");
String s2 = s1.toUpperCase();
String s3 = s1.toLowerCase();
System.out.println(s1 == s2); // false
System.out.println(s1 == s3); // true
```

```java
String s1 = "hello";
String s2 = s1.toString();
String s3 = s1.toLowerCase();
String s4 = s1.toUpperCase();
System.out.println(s1 == s2); // true
System.out.println(s1 == s3); // true
System.out.println(s1 == s4); // false
```

## ‚úÖ **StringBuffer or StringBuilder**
```java
StringBuffer sb = new StringBuffer("Hello");
sb.append(" World!");
System.out.println(sb);
```
* `StringBuffer sb = new StringBuffer("Hello");` creates a new object in heap. 
* Since StringBuffer class is mutable therefore `sb.append(" World!");` will append `" World!‚Äù` to same object. 
* Therefore, `System.out.println(sb);` will print out `"Hello World!‚Äù.`
